# Anime Realtime Conversion via Deep Learning

## 課題

シェーダーだけによるアニメ変換（Anime Cel パレット）は以下の限界がある：

- ピクセル単体では「肌なのか背景なのか」が判断できない
- 影の方向・形状（嘘カゲ）を意味的に計算できない
- 色域ごとの処理（肌・髪・服を分離）ができない

これらを解決するには画像の意味を理解する Deep Learning が必要。

## アプローチ候補

### セグメンテーション + 領域別処理
- 肌・髪・服・背景を分離して、それぞれに適した色量子化・影処理を適用
- 領域境界でのエッジ検出精度も向上できる

### 法線推定
- 1枚の画像から擬似的な法線マップを推定
- 影方向の計算、嘘カゲの自動生成が可能になる

### AnimeGAN 系のエンドツーエンド変換
- AnimeGAN v2/v3 は PC GPU で 30fps 前後の実績あり
- 実写→アニメ変換をモデルが一括で処理する

## ブラウザ上での現実的な構成

```
入力フレーム
  → 低解像度にダウンスケール（例：320x180）
  → ONNX Runtime Web（WebGL バックエンド）でアニメ変換
  → WebGL でアップスケール + エッジ強調
  → 既存の CRT / Scanline シェーダーと合成
  → 出力
```

既存のシェーダーパイプラインの前段に ONNX を挟む形なので、
CRT・Phosphor Dot 等のエフェクトとの組み合わせも維持できる。

## プラットフォーム別の見通し

| 環境 | 見通し |
|---|---|
| PC ブラウザ（WebGL ONNX） | 軽量モデルなら動作可能、640x360 は重い |
| iOS（Neural Engine） | ネイティブなら十分な速度が期待できる |
| Android | デバイス依存、mid-range 以上なら可能性あり |

## 優先度・時期

- v2.x 以降のネタとして温める
- まず ONNX Runtime Web で既存の AnimeGAN 軽量モデルを試験動作させることが最初のステップ
